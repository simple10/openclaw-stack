# Always Multi-Claw Architecture (Phase 1: Core)

## Context

The current system has two modes: single-claw (docker-compose.override.yml, `openclaw-gateway`) and multi-claw (docker-compose.multi.yml, `openclaw-<name>`). This dual-mode creates bugs in tunnel route management (doesn't know about the default claw), complicates scripts (12+ files hardcode `openclaw-gateway`), and makes per-claw customization awkward (where do main claw's AGENTS.md/SOUL.md files go?).

**This refactor eliminates the distinction.** There's always multi-claw — it just might be running one. Every claw lives in `deploy/openclaws/<name>/`, gets the same treatment, and uses the same code paths.

**Phase 1 (this PR):** Core architecture — directory layout, compose generation, deploy scripts, tunnel setup.
**Phase 2 (future):** Bash scripts + CLI updates (replace hardcoded `openclaw-gateway` with dynamic resolution).
**Phase 3 (future):** Playbook + documentation updates.

---

## Key Design Decisions

**Container name = `openclaw-<dirname>`** — derived from the directory name, not a config var. `main-claw/` → container `openclaw-main-claw`.

**`_defaults/` layering** — `deploy/openclaws/_defaults/` holds shared templates (openclaw.json, models.json). At deploy time: copy `_defaults/*` into each claw's config dir, then overlay claw-specific files on top.

**Generated `docker-compose.override.yml`** — `openclaw-multi.sh generate` writes `docker-compose.override.yml` on the VPS (replacing the static one from the repo). This means ALL existing `docker compose up -d` commands auto-load it without needing `-f` flags. The upstream `docker-compose.yml` stays unmodified; its `openclaw-gateway` service gets disabled via `profiles: [disabled]`.

**VPS layout: `/home/openclaw/instances/<name>/`** — full per-claw isolation:

```
/home/openclaw/
  openclaw/                          # shared source + compose
    docker-compose.yml               # upstream (unchanged)
    docker-compose.override.yml      # GENERATED by openclaw-multi.sh
    .env                             # generated per-instance vars
    scripts/                         # shared scripts
    deploy/                          # shared deploy files
  instances/
    main-claw/
      .openclaw/                     # gateway config (openclaw.json, models.json, agents/...)
        workspace/
        credentials/
        logs/
        backups/
      sandboxes-home/                # agent sandbox home dirs
      docker/                        # nested Docker daemon storage
    test-claw/
      .openclaw/
      sandboxes-home/
      docker/
```

## Implementation

### 1. Restructure `deploy/openclaws/`

**Create `deploy/openclaws/_defaults/`:**

- Move `deploy/openclaw.json` → `deploy/openclaws/_defaults/openclaw.json`
- Move `deploy/models.json` → `deploy/openclaws/_defaults/models.json`

**Create `deploy/openclaws/main-claw/config.env`** (tracked in git):

```bash
# Main claw — the default OpenClaw instance.
# Inherits all settings from openclaw-config.env.
# Per-claw customizations (AGENTS.md, SOUL.md, etc.) go in this directory.
OPENCLAW_INSTANCE_ID=main-claw
```

**Update `.gitignore`:**

```gitignore
deploy/openclaws/*
!deploy/openclaws/_defaults/
!deploy/openclaws/_example/
!deploy/openclaws/main-claw/
```

**Delete `deploy/docker-compose.override.yml`** — replaced by generated output.

### 2. Rewrite `deploy/scripts/openclaw-multi.sh`

Remove all single-vs-multi branching. Always generate compose for all discovered claws.

**`discover_instances()`** — unchanged logic (finds dirs with config.env, excludes `_` prefix). Now always finds at least `main-claw`.

**`assign_ports()`** — unchanged (alphabetical sort, base + idx). `main-claw` sorts first alphabetically, gets idx=0 → ports 18789/6090.

**`generate_compose()`** — generates `docker-compose.override.yml` (not multi.yml) with:

- `x-openclaw-base` anchor (all shared config from current override.yml: Sysbox, tmpfs, security, healthcheck, logging)
- Per-claw services named `openclaw-<dirname>`
- Volumes using `/home/openclaw/instances/<name>/` paths:

  ```yaml
  volumes:
    - ./scripts/entrypoint-gateway.sh:/app/scripts/entrypoint-gateway.sh:ro
    - /home/openclaw/instances/<name>/sandboxes-home:/home/node/sandboxes-home
    - /home/openclaw/instances/<name>/docker:/var/lib/docker
    - ./deploy:/app/deploy:ro
    - /home/openclaw/instances/<name>/.openclaw:/home/node/.openclaw
  ```

- Upstream `openclaw-gateway` and `openclaw-cli` disabled via `profiles: [disabled]`
- Network: `openclaw-gateway-net: external: true`

**`cmd_start()`** — just `docker compose up -d` (auto-loads generated override.yml).

**`cmd_stop()`** — just `docker compose down`.

**`cmd_tunnel_config()`** — shows routes for ALL discovered claws (including main-claw). With `--apply`, delegates to `cf-tunnel-setup.sh setup-routes`.

**`generate_env()`** — builds `.env` with shared vars + per-instance vars for all claws.

### 3. Update `deploy/scripts/setup-infra.sh`

Replace current directory creation with instance-based layout:

```bash
# For each claw:
mkdir -p /home/openclaw/instances/<name>/{.openclaw/{workspace,credentials,logs,backups},sandboxes-home,docker}
chown -R openclaw:openclaw /home/openclaw/instances/<name>
```

Remove old single-instance paths (`/home/openclaw/.openclaw/`, `/home/openclaw/sandboxes-home/`, `/home/openclaw/openclaw/data/docker`).

### 4. Update `deploy/scripts/deploy-config.sh`

Replace single/multi branching with one flow. For each claw:

1. Copy `deploy/openclaws/_defaults/*` to staging area
2. Copy `deploy/openclaws/<name>/*` on top (excluding `config.env`)
3. Load layered env vars: `openclaw-config.env` → `<name>/config.env`
4. Template all files (substitute `{{VAR}}` placeholders)
5. Deploy to `/home/openclaw/instances/<name>/.openclaw/`

Also deploys per-agent `models.json` to each claw's `.openclaw/agents/` subdirectories.

Remove `SHARED_ONLY` and `INSTANCE_NAME` modes — there's now just one mode.

### 5. Update `deploy/scripts/cf-tunnel-setup.sh`

**`setup-routes`** — replace three-branch logic (multi / single-instance / default) with one flow:

1. Discover all claws via `discover_instances()` (same as openclaw-multi.sh)
2. For each claw: load layered config, get domain + ports
3. Build ingress rules for all claws
4. PUT tunnel config + create DNS CNAMEs

No more special handling for "default claw" or "single-instance mode."

### 6. Update `deploy/entrypoint-gateway.sh`

Minimal changes — the entrypoint already works with `/home/node/.openclaw` which is bind-mounted. Just verify:

- Lock file cleanup works with the bind mount path
- Permission fixes target the right directories
- No hardcoded paths outside of `/home/node/` (container-internal)

### 7. Update logrotate config

`deploy/logrotate-openclaw`:

```
/home/openclaw/instances/*/.openclaw/logs/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
}
```

### 8. Update backup script

`deploy/backup.sh` (or equivalent):

- Backup target: `/home/openclaw/instances/` (covers all claws)
- Each claw's `.openclaw/backups/` dir holds its own backups
- Shared source (`/home/openclaw/openclaw/`) doesn't need backup (it's deployable)

## What This Looks Like

**Single claw (default fresh deploy):**

```
deploy/openclaws/
  _defaults/openclaw.json, models.json
  main-claw/config.env
  _example/config.env
```

VPS: `openclaw-main-claw` container at 18789/6090.

**Adding a claw:**

```
deploy/openclaws/
  _defaults/openclaw.json, models.json
  main-claw/config.env
  test-claw/config.env          # OPENCLAW_DOMAIN=test-openclaw.ventureunknown.com
  _example/config.env
```

VPS: `openclaw-main-claw` at 18789/6090, `openclaw-test-claw` at 18790/6091.

**Per-claw customization (future):**

```
deploy/openclaws/
  _defaults/openclaw.json, models.json, AGENTS.md
  main-claw/config.env, AGENTS.md       # overrides default AGENTS.md
  test-claw/config.env                  # uses default AGENTS.md
```

## Verification

1. `openclaw-multi.sh list` — shows main-claw (and any extras)
2. `openclaw-multi.sh generate` — produces docker-compose.override.yml with all claws
3. `cf-tunnel-setup.sh setup-routes` — configures tunnel routes for all claws
4. `openclaw-multi.sh tunnel-config` — shows correct routes with correct ports
5. VPS directory structure: `/home/openclaw/instances/main-claw/{.openclaw,sandboxes-home,docker}`
6. `docker compose up -d` on VPS — starts all claw containers
7. `docker compose ps` — shows `openclaw-main-claw` (not `openclaw-gateway`)
