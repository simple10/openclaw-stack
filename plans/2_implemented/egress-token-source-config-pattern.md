# Plan: Auto-Generate Ephemeral Tokens + source-config.sh --force

## Context

Tokens like `EGRESS_PROXY_AUTH_TOKEN` are currently generated manually during onboarding (`openssl rand -hex 32`) and written to `.env`. This requires extra tool calls from Claude, puts build-infrastructure logic in playbooks, and creates inconsistency ‚Äî some tokens (gateway, sandbox registry) are auto-generated by pre-deploy while others need manual generation.

**New pattern:** Claude edits `stack.yml` to enable features. `pre-deploy.mjs` auto-generates any missing tokens via `resolveAutoToken()` (explicit .env value > cached in stack.json > new random). Scripts and playbooks get resolved values exclusively from `source-config.sh`.

---

## 1. `source-config.sh` ‚Äî Auto-Run Pre-Deploy + `--force`

### `deploy/host/source-config.sh`

In the local context block, after determining `_STACK_ENV` path but before the "not found" error:

```bash
# Auto-run pre-deploy if stack.env missing or --force requested (local only)
if [ "$OPENCLAW_CONTEXT" = "local" ]; then
  _FORCE="${1:-}"
  if [ ! -f "$_STACK_ENV" ] || [ "$_FORCE" = "--force" ]; then
    echo "‚Üí Running pre-deploy to generate stack.env..." >&2
    (cd "$REPO_ROOT" && npm run pre-deploy) >&2
  fi
fi
```

The existing error block stays as-is ‚Äî if pre-deploy fails, stack.env still won't exist and the script errors with the existing message.

### `scripts/lib/source-config.sh` (thin wrapper)

Pass args through:

```bash
source "$_LIB_DIR/../../deploy/host/source-config.sh" "$@"
```

## 2. `pre-deploy.mjs` ‚Äî Auto-Generate `EGRESS_PROXY_AUTH_TOKEN`

### In `computeDerivedValues()`

```javascript
if (stack.egress_proxy) {
  stack.egress_proxy.auth_token = resolveAutoToken(
    stack.egress_proxy.auth_token,
    "stack.egress_proxy.auth_token",
    previousDeploy
  );
}
```

### In `generateStackEnv()`

```javascript
if (stack.egress_proxy) {
  lines.push(`STACK__STACK__EGRESS_PROXY__AUTH_TOKEN=${formatEnvValue(stack.egress_proxy.auth_token)}`);
}
```

## 4. `stack.yml.example` ‚Äî Use `${VAR:-}` for auto-gen tokens

```yaml
  egress_proxy:
    port: 8787
    auth_token: ${EGRESS_PROXY_AUTH_TOKEN:-}  # auto-generated if empty
    log_level: info
```

```yaml
  sandbox_registry:
    port: 5100
    token: ${SANDBOX_REGISTRY_TOKEN:-}        # auto-generated if empty
```

The `:-` default syntax means empty .env values resolve to empty string (triggering auto-gen) instead of logging "unresolved env var" warnings.

## Verification

1. **Token auto-gen:** Remove EGRESS_PROXY_AUTH_TOKEN from .env, run pre-deploy ‚Üí token appears in stack.json and stack.env
2. **Token stability:** Run pre-deploy twice ‚Üí same token (cached in stack.json)
3. **Explicit wins:** Set `EGRESS_PROXY_AUTH_TOKEN=mytoken` in .env ‚Üí pre-deploy uses it
4. **source-config.sh auto-run:** Delete .deploy/, `source scripts/lib/source-config.sh` ‚Üí runs pre-deploy, vars available
5. **--force:** Edit stack.yml, `source scripts/lib/source-config.sh --force` ‚Üí re-runs pre-deploy, picks up changes
6. **Backwards compat:** Existing .env with EGRESS_PROXY_AUTH_TOKEN still works
